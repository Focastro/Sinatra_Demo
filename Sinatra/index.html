Historia

Es un framework de desarrollo web Ruby. Diseñado y desarrollado por Blade Mizerany en el año 2007 en California.

...

Este framework fue construido gracias a que otras empresas dieron un poquito de aporte para que se hiciera posible, tanto en apoyo financiero como emocional, las cuales son Heroku, Github y Engine Yard, y ahora por Travis CI, entre otras.

Personajes destacados

  • Blake Mizerany
  • Ryan Tomayko
  • Simon Rozet
  • Konstantin Haase

Estado actual

La ultima version es 1.2.6

Aspectos interesantes y diferenciadores

  • No incluye montones de librerías y helpers.
  • Tiene lo mínimo para hacer una aplicación web.
  • No tiene ORMs asociados, wrappers complejos para las vistas.
  • No hay convenciones predefinidas.
  • No hay una estructura específica para tu código.

Ventajas

  • Multiplataforma.
  • Es muy liviano.
  • Proyectos simples.
  • Es muy sencillo.
  • Flexible.
  • Mejor rendimiento que Rails
  • No hay magia oscura involucrado

Desventajas

  • Proyectos grandes.
  • Ausencia de conexiones con magia mistica.
  • Solo tiene lo basico
  • No es un framework MVC.
  • No es popular como rails.
  • Tiene apoyo de la comunidad pobre.

Quienes lo utilizan

  • The movie DB.
  • Github.
  • Nesta CMS.
  • Blitz IO.

Para que tipos de proyectos lo recomienda

Que se necesita

  1. $ gem install sinatra
  2. $ touch [nombreApp].rb
  3. Agrega "require 'sinatra'" al [nombreApp].rb
  4. $ ruby [nombreApp].rb
  5. http://localhost:4567
